RediSearch

Search

The indexes resource is used to execute search commands FT.CREATE, FT.INFO, FT.DROP, and FT.ADD.

Creating an index

A POST request is used to create an index.

Request fields
Path Type Description

index

String

index name

maxTextFields

Boolean

see MAXTEXTFIELDS

noOffsets

Boolean

see NOOFFSETS

noHL

Boolean

see NOHL

noFields

Boolean

see NOFIELDS

noFreqs

Boolean

see NOFREQS

stopWords

Array

see STOPWORDS

fields

Array

see SCHEMA

fields[].name

String

field name

fields[].sortable

Boolean

allows the user to later sort the results by the value of this field

fields[].noIndex

Boolean

field will not be indexed

fields[].type

String

text, numeric, or geo
Example request
POST /indexes HTTP/1.1
Content-Type: application/json
Host: localhost:8080
Content-Length: 388

{
  "index" : "testIndex",
  "maxTextFields" : false,
  "noOffsets" : false,
  "noHL" : false,
  "noFields" : false,
  "noFreqs" : false,
  "stopWords" : [ "a", "i", "the" ],
  "fields" : [ {
    "name" : "field1",
    "sortable" : false,
    "noIndex" : false,
    "type" : "text"
  }, {
    "name" : "field2",
    "sortable" : true,
    "noIndex" : false,
    "type" : "numeric"
  } ]
}
Example response
HTTP/1.1 200 OK
Content-Type: application/json;charset=UTF-8
Content-Length: 4

true

Dropping an index

A DELETE request is used to drop an index.

Request fields
Path Type Description

missingOk

Boolean

Do not throw error if index does not exist
Example request
DELETE /indexes/testIndex HTTP/1.1
Content-Type: application/json
Host: localhost:8080
Content-Length: 24

{
  "missingOk" : true
}
Example response
HTTP/1.1 200 OK
Content-Type: application/json;charset=UTF-8
Content-Length: 4

true

Adding documents

A POST request is used to drop an index.

Request fields
Path Type Description

documents

Array

An array of documents to add

documents[].fields

Object

Document fields

documents[].id

String

Document id

documents[].score

Number

Document score, between 0.0 and 1.0

documents[].payload

String

Document payload

documents[].fields

Object

Document fields

options

Object

Add options

options.noSave

Boolean

Do not save document fields

options.language

String

Language of the documents

options.replacementPolicy

String

NONE, FULL, or PARTIAL
Example request
POST /indexes/testIndex/documents HTTP/1.1
Content-Type: application/json
Host: localhost:8080
Content-Length: 556

{
  "documents" : [ {
    "id" : "doc1",
    "score" : 1.0,
    "payload" : "cGF5bG9hZDE=",
    "fields" : {
      "field1" : "malibu"
    }
  }, {
    "id" : "doc2",
    "score" : null,
    "payload" : "cGF5bG9hZDI=",
    "fields" : {
      "field1" : "santa monica",
      "field2" : 123.321
    }
  }, {
    "id" : "doc3",
    "score" : null,
    "payload" : null,
    "fields" : {
      "field1" : "santa cruz",
      "field2" : 122.0
    }
  } ],
  "options" : {
    "noSave" : false,
    "language" : "english",
    "replacementPolicy" : "NONE"
  }
}
Example response
HTTP/1.1 200 OK

Deleting documents

A DELETE request is used to drop an index.

Example request
DELETE /indexes/testIndex/documents HTTP/1.1
Content-Type: application/json
Host: localhost:8080
Content-Length: 18

[ "doc1", "doc2" ]
Example response
HTTP/1.1 200 OK

Searching

A POST request is used to search an index.

Request fields
Path Type Description

query

String

Query string

withScores

Boolean

Include document scores in search results

withPayloads

Boolean

Include document payloads in search results

language

String

Language to be used for search results

sortBy

String

Name of field to be used for sorting search results

sortAscending

Boolean

Sort results in ascending order

verbatim

Boolean

Verbatim

noStopWords

Boolean

No stop words

noContent

Boolean

No content

limit

Number

Number of search results

offset

Number

Start index for search results
Example request
POST /indexes/testIndex/search HTTP/1.1
Content-Type: application/json
Host: localhost:8080
Content-Length: 264

{
  "query" : "santa @field2:[120 124]",
  "offset" : 0,
  "limit" : 30,
  "sortBy" : "field2",
  "sortAscending" : true,
  "language" : "english",
  "noContent" : false,
  "noStopWords" : true,
  "verbatim" : true,
  "withPayloads" : true,
  "withScores" : true
}
Example response
HTTP/1.1 200 OK
Content-Type: application/json;charset=UTF-8
Content-Length: 353

Autocomplete

The suggestions resource is used to execute suggestion commands FT.SUGADD and FT.SUGGET.

Adding suggestions

A POST request is used to add suggestions.

Request fields
Path Type Description

suggestions

Array

list of suggestions to add

suggestions[].string

String

suggestion string

suggestions[].payload

String

suggestion payload

suggestions[].score

Number

suggestion score

increment

Boolean

increment suggestion score
Example request
POST /suggestions/suggestionIndex HTTP/1.1
Content-Type: application/json
Host: localhost:8080
Content-Length: 281

{
  "suggestions" : [ {
    "string" : "suggestion1",
    "score" : null,
    "payload" : null
  }, {
    "string" : "suggestion2",
    "score" : null,
    "payload" : null
  }, {
    "string" : "suggestion3",
    "score" : null,
    "payload" : null
  } ],
  "increment" : false
}
Example response
HTTP/1.1 200 OK

Getting suggestions

A GET request is used to get suggestions.

Request parameters
Parameter Description

prefix

Prefix

fuzzy

Fuzzy matching

max

Max number of matches

withPayloads

Include payloads with matches

withScores

Include scores with matches
Example request
GET /suggestions/suggestionIndex?prefix=sug HTTP/1.1
Host: localhost:8080
Example response
HTTP/1.1 200 OK
Content-Type: application/json;charset=UTF-8
Content-Length: 209

[ {

}, {

} ]
